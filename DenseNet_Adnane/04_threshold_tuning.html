Dans un modle de classification binaire, comme celui que nous utilisons pour dtecter la pneumonie partir de radiographies pulmonaires, la sortie du modle correspond une probabilit indiquant si une image appartient la classe PNEUMONIA.

Par dfaut, le modle utilise un seuil de 0.5 pour transformer cette probabilit en prdiction :

Si la probabilit  0.5 : l'image est classe comme PNEUMONIA

Si la probabilit < 0.5 : l'image est classe comme NORMAL

Le seuil de 0.5 nest pas toujours optimal. En ajustant ce seuil, on peut faire varier lquilibre entre la prcision et le rappel :

Un seuil plus bas favorise la dtection des cas de pneumonie (rappel lev), mais augmente le risque de fausses alertes.

Un seuil plus lev favorise la certitude dans les cas positifs (prcision leve), mais risque de manquer des cas rels.

Dans un contexte mdical, il est important dadapter le seuil selon les risques associs aux faux positifs ou faux ngatifs.

Dans ce notebook, nous avons :

Charg les probabilits prdites par notre modle sur lensemble de test, ainsi que les vraies tiquettes.

Calcul les mtriques (prcision, rappel, F1-score) pour plusieurs seuils allant de 0.1  0.9.

Stock les rsultats dans un DataFrame pour les analyser.

Visualis lvolution des mtriques en fonction du seuil  laide dun graphique. Ce graphique montre comment chaque mtrique varie en fonction du seuil.

thresholds = np.arange(0.3, 0.61, 0.05)
results = []

for thresh in thresholds:
    y_pred = (y_prob >= thresh).astype(int)
    precision = precision_score(y_true, y_pred)
    recall = recall_score(y_true, y_pred)
    f1 = f1_score(y_true, y_pred)
    results.append({
        'threshold': thresh,
        'precision': precision,
        'recall': recall,
        'f1_score': f1
    })

df_results = pd.DataFrame(results)
