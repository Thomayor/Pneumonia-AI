Le fine-tuning consiste :

Dbloquer certaines couches du modle pr-entran (ici DenseNet121) qui taient geles lors de lentranement initial.

Rentraner ces couches sur notre propre dataset mdical pour que le modle affine ses reprsentations et sadapte aux spcificits des radiographies pulmonaires.

Cela permet daugmenter la capacit dadaptation du modle, notamment pour des dtails visuels trs spcifiques notre cas dusage (ex. : motifs lis la pneumonie).

Chargement du modle DenseNet121 dj entran (avec uniquement la tte personnalise).

Dblocage progressif des couches profondes (haut du backbone).

Rcompilation du modle avec un taux dapprentissage plus faible (pour viter daltrer brutalement les poids pr-entrans).

Rentraner le modle sur les donnes dentranement et validation pendant quelques poques.

Enregistrement du nouveau modle fine-tun pour une valuation ultrieure.

Pourquoi c'est important ? :

Le modle peut mieux capturer les motifs spcifiques aux radiographies pulmonaires.

# Compiler avec un learning rate faible
model.compile(
    optimizer=Adam(learning_rate=1e-5),
    loss='binary_crossentropy',
    metrics=['accuracy']
)

# Callbacks
early_stop = EarlyStopping(monitor='val_loss', patience=2, restore_best_weights=True)
checkpoint = ModelCheckpoint(finetuned_model_path, save_best_only=True)

# Entranement 
history_ft = model.fit(
    train_ds,
    validation_data=val_ds,
    epochs=5,
    callbacks=[early_stop, checkpoint]
)
